# Code Review Checklist

## 適用範圍
Phase 3 Isaac Sim / Omniverse Kit Extension 專案開發期間的自我審查清單。
每次完成一個功能單元後，提交前逐項確認。

---

## 1. 層次職責確認

> 確認程式碼放在正確的架構層次。

- [ ] 此程式碼是放在 `core/` 還是 `extension/`？職責是否正確？
- [ ] `core/` 內是否出現任何 Isaac Sim / Omniverse 的 import？（不應該出現）
- [ ] `extension/` 內是否有商業邏輯應該移到 `core/`？
- [ ] `extension/` 是否只做薄層橋接（呼叫 API、轉換資料、注入 `core/`）？
- [ ] 新增的類別或函式，職責邊界是否清楚？有沒有一個函式做了兩件不同的事？

---

## 2. 依賴注入確認

> 確認 UI 與資料解耦，以及抽象介面的正確使用。

- [ ] UI 元件是否透過注入取得資料，而非直接存取資料來源？
- [ ] 是否有提供開發期間可用的假資料（Stub / Fake）？
- [ ] 可替換的元件（例如 `ScriptController` / `ModelController`）是否有定義抽象介面（`ControllerBase`）？
- [ ] 高層邏輯是否只依賴抽象介面，而非具體實作類別？

---

## 3. Unit Test 確認

> 依照 `unit-test-rules.md` 的判斷規則執行。

- [ ] 此函式是否需要寫 Unit Test？（參照判斷規則）
- [ ] 如果需要，Test 是否在實作前已撰寫（TDD）？
- [ ] 測試案例是否涵蓋正常路徑、邊界值、錯誤情境？
- [ ] 涉及外部依賴（Isaac Sim API、MQTT、檔案系統）的函式，是否有正確使用 Mock？
- [ ] 所有現有測試是否仍然通過？

---

## 4. 錯誤處理合理性

> 確認錯誤處理存在且合理。

**必要條件：**
- [ ] 是否有明確的 `except` 捕捉對象，避免裸露的 `except:` 或無理由的 `except Exception:`？
- [ ] 捕捉到例外後是否有 log 輸出（至少包含錯誤類型與發生位置）？
- [ ] 是否有靜默吞掉錯誤的狀況（catch 後什麼都不做）？

**視情況判斷：**
- [ ] 如果是可恢復的錯誤，是否有 fallback 或 retry 邏輯？
- [ ] 如果是不可恢復的錯誤，是否有明確的狀態重置或關閉流程？
- [ ] 錯誤是否需要向上傳遞給呼叫者？若在此層消化，理由是否合理？

---

## 5. 效能考量

> 特別針對 Update 迴圈內執行的程式碼。

- [ ] 此函式是否在 Update 迴圈內被呼叫？
- [ ] 是否有 O(n²) 以上的迴圈結構？若有，資料規模是否可控？
- [ ] 是否有不必要的重複計算（應快取的結果沒有快取）？
- [ ] 是否在 Update 迴圈內做 USD 寫入？若有，是否確認只在數值變化時才寫入？
- [ ] 是否在 Update 迴圈內做記憶體配置（建立新物件、list comprehension）？若有，是否有必要？

---

## 6. 執行緒安全

> 避免競態條件與鎖死。

- [ ] 此函式是否會在多個執行緒中被呼叫？
- [ ] 跨執行緒的共享資料是否使用 `queue.Queue` 或 `threading.Lock` 保護？
- [ ] USD 寫入是否確保只在主執行緒執行？
- [ ] 是否有可能發生鎖死（兩個執行緒互相等待對方釋放鎖）？
- [ ] MQTT callback 中是否有直接操作共享資料？（應改用 queue buffer）

---

## 7. 命名與風格

> 遵循 PEP 8 與專案規範。

- [ ] 命名是否遵循 PEP 8（`snake_case` 函式與變數、`PascalCase` 類別）？
- [ ] 程式碼註解是否使用繁體中文，技術術語保持英文？
- [ ] 函式長度是否合理（建議單一函式不超過 30 行）？
- [ ] 是否有未使用的 import 或變數？
- [ ] `# REFACTOR:` 的 inline 標記是否已記錄需要日後處理的技術債？

---

## 快速決策流程

```
新增或修改程式碼
        ↓
是否在 core/ 還是 extension/？
        ↓
職責是否正確？ → 否 → 移到正確層次
        ↓ 是
是否需要 Unit Test？（參照 unit-test-rules.md）
        ↓ 是
Test 是否已撰寫且通過？ → 否 → 補寫 Test
        ↓ 是
逐項確認 Checklist 4～7
        ↓
提交
```
